## 2024-11-20 - Eager f-string Evaluation in Python Logging
**Learning:** Python f-strings in `_logger.debug(f"...")` calls are evaluated eagerly before the log level is checked. In high-frequency network loops (like `EventThread.run` and `_processPayload` in `cipclient.py`), string formatting overhead causes measurable CPU waste even when `logging.DEBUG` is disabled. Furthermore, manually testing lazy `%` interpolation (e.g. `_logger.debug("...", val)`) showed it's slower than explicitly checking `if _logger.isEnabledFor(...)`.
**Action:** When finding f-strings inside `_logger.debug()` calls within hot paths, do not simply convert them to lazy `%` interpolation. Wrap them in explicit `if _logger.isEnabledFor(...)` blocks for maximum performance.
